很多人第一次把 GitHub Pages 绑定自定义域名时，会觉得只要页面能打开就算完成了。实际上，github.io 域名和你自己的域名可能走的不是同一份内容，尤其在有 Cloudflare 代理时更容易出现这种分叉。

一个常见现象是，本地预览没问题，github.io 也正常，但自定义域名显示旧页面、白屏，或者加载的资源和主站不一致。这通常不是前端本身的问题，而是部署链路和缓存没有统一。

稳妥的做法是三件事同时确认：GitHub Pages 的构建来源、仓库里的 CNAME 文件、自定义域名的 DNS 与代理状态。只检查其中一项，经常会留下半通不通的状态。

对个人站来说，排查这类问题最重要的不是猜，而是直接抓线上返回的 HTML 和资源链接。只要看见两个域名返回了不同的入口文件，问题基本就已经锁定了。
